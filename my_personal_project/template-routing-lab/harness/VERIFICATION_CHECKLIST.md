# 验证清单

## 1. 合同层与 Schema 改动
- 类型可以通过 `tsc --noEmit` 编译检查。
- 模板元数据 schema 覆盖来源类型、标签、别名和路由相关字段。
- 检索结果和路由结果都包含机器可读的原因。
- 日志 schema 不包含密钥或不必要的敏感字段。

## 2. 模板注册表改动
- 每个模板都有稳定的 `id`。
- 每个模板都有完整元数据。
- 每个模板都指向有效的本地内容来源。
- 已注册种子模板可通过别名和分类标签被召回。
- 模板元数据与实际几何意图一致。

## 3. Prompt 规范化改动
- 相同输入总能规范化为相同输出。
- 大小写、空白、标点噪声处理一致。
- 别名抽取对固定测试用例表现符合预期。
- 负向关键词在冲突场景下能产生可见信号。

## 4. 检索与排序改动
- 已知 prompt 能召回预期模板。
- 明显无关的模板不会排在相关模板之前。
- 排序输出包含分数拆解或命中信号。
- 所有打分规则变化都有测试覆盖。
- `car`、`castle` 这类场景不会让动物模板高分误命中。

## 5. 路由改动
- 高置信 prompt 能按预期走 `reuse`。
- 中置信且存在可编辑模板的 prompt 能走 `adapt`。
- 低置信或明显不兼容的 prompt 能走 `free_generate`。
- `morph` 模式下，`rebuildSuitability` 会对排序或路由产生真实影响。
- 所有路由结果都带明确原因。

## 6. 日志与反馈改动
- 检索事件能记录 prompt 摘要、候选模板、分数和最终路由。
- 日志不含密钥、令牌或运行时凭据。
- 反馈事件是可选、低打扰的。
- 反馈数据不会直接修改线上路由行为。
- 存储下来的事件结构足以支持离线分析。

## 7. 生成式扩展挂钩
- 生成式候选模板和正式模板注册表是分离的。
- 所有生成式候选在提升为正式模板前都需要校验。
- 校验至少覆盖重复性、元数据完整度和基础质量门槛。
- 生成式候选默认不能直接生效。

## 8. Review 就绪性
- 改动中至少包含一个端到端的 decision trace 示例。
- 新增阈值或打分规则有文档说明。
- 修复误判时补了回归测试。
- 不需要通读整个 legacy 项目，也能理解本模块的行为。
