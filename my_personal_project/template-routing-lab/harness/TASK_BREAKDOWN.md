# 任务拆解草案

## 任务 1：定义合同层与目录结构
### 目标
先把模板、检索、路由、日志四类核心边界定义清楚，避免后续各自写各自的类型。

### 交付物
- 模板元数据类型
- 检索结果类型
- 路由结果类型
- 日志事件类型
- 新模块目录结构

### 说明
- 这是第一优先级任务，后续任务不得绕开这层类型约束自行扩展。

## 任务 2：注册首批种子模板
### 目标
把 `Eagle`、`Cat`、`Rabbit`、`Twins` 从代码级预设转成正式模板资产。

### 交付物
- 4 个种子模板的注册记录
- 每个模板的手工元数据
- 指向种子生成器的几何来源引用
- 一份“v1 什么算模板”的说明

### 说明
- v1 元数据必须人工编写，不用自动生成。
- 至少要包含别名、负向关键词、可编辑部位。

## 任务 3：实现 prompt 规范化与规则召回
### 目标
完成模板检索第一层：从原始 prompt 提取可用于召回模板的稳定信号。

### 交付物
- prompt 规范化模块
- 基于关键词和别名的召回逻辑
- 基于负向关键词的冲突标记
- 候选模板与命中证据输出

### 说明
- 首版优先确定性和可解释性，不追求过宽召回。

## 任务 4：实现打分与排序模块
### 目标
对候选模板基于元数据和 prompt 信号进行透明打分。

### 交付物
- 类别、风格、形状、复杂度、重建适配性打分规则
- 候选模板排序输出
- decision trace 结构
- 路由阈值常量

### 说明
- 每一项分数加减都必须可以解释。

## 任务 5：实现路由决策层
### 目标
把排序结果稳定地映射到 `reuse / adapt / free_generate`。

### 交付物
- 路由决策模块
- `create` 与 `morph` 的差异化策略
- 路由理由输出
- 低置信场景的兜底逻辑

### 说明
- `morph` 模式下，应提高 `editableParts` 和 `rebuildSuitability` 的权重。

## 任务 6：实现日志与轻量反馈事件采集
### 目标
建立可离线分析的结构化事件，不做在线学习。

### 交付物
- 检索事件 schema
- 路由事件 schema
- 轻量反馈事件 schema
- 本地持久化设计或适配接口

### 说明
- 反馈数据只做评估输入，不回写运行时策略。

## 任务 7：建立验证样例与测试集
### 目标
让模块具备工程可审查性，而不是只能靠人工试。

### 交付物
- 固定 prompt fixture 集
- 排序测试
- 路由测试
- 日志结构测试
- 误判回归测试

### 说明
- 至少覆盖 `bird`、`cute rabbit`、`double birds`、`castle`、`car`、`robot` 等场景。
